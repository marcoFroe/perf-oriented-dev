### Parameters
Following is a description of the parameters needed for each program. I considered the programs as black boxes and did not change any source code files.

- *delannoy*: problem size between 1 and 22
- *filegen*: needs instruction on how many directories and files to generate and at what size -- `./filegen <num_directories> <num_files_per_directory> <min_file_size> <max_file_size>`

- *mmul*: no arguments required
- *nbody*: no arguments required
- *qap*: path to the problem data

## Exercise 2 - Experiments
My experimentation script consists of two programs: 

***experiment_generator.py***: 

This script reads experiment configuration from a provided `json` file and executes the desired program with all combinations of requested parameters. In the configuration file also the measurement tools with the metrics of interest are being defined. The script than executes each and every combination of parameters and measurement tools as a sub-process. The raw results of the measurement tools are stored in one `csv` file per tool. The output of the programs end up in the standard log file generated by `slurm`.

 The working procedure is to create the configuration file and schedule this script via `sbatch` on LCC3, for a personal machine ditch the `sbatch` and just run the script.

Currently, the tool supports `/usr/bin/time` and `perf stat` as measurement tools, additional tools should be easy to be added.

To make my life easier with the many different example executables I created a bash script that looks for `.slurm` files starting from a parent directory and runs them with `sbatch`.

***automated_evaluator.py***:

This script finds all `csv` files starting a given parent directory and computes the average for all found configuration inside this file. The results are added as comments to the end of the file. The plan is to extend this script to automatically generate plots, but for that I need more information on what metrics are of interest and in what way to display them.
